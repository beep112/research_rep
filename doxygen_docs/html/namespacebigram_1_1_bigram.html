new version of get_batch that allows for model to be loaded to gpus

The previous version of get_batch in the Tensor_Prac doesn't have the ability for the GPU to actualy be used, even if the device is present. So this new version will take advantage of that hardware.

117 The previous version of get_batch in the Tensor_Prac doesn't have the ability
118 for the GPU to actualy be used, even if the device is present. So this new
119 version will take advantage of that hardware.
120
121 @param split: string either 'train' or 'val' which is used to pick data to batch with
122 @param train_data: the data that is split for training
123 @param val_data: the data that is split for validation
124 @return Tuple(Tensor of inputs, Tensor or targets)
125 """
126 # generate a small batch of data of inputs x and targets y
127 data = train_data if split == "train" else val_data
128 ix = torch.randint(len(data) - block_size, (batch_size,))
129 x = torch.stack([data[i : i + block_size] for i in ix])
130 y = torch.stack([data[i + 1 : i + block_size + 1] for i in ix])
131 # IMPORTANT: This is the part that is needed for the GPU
132 x, y = x.to(device), y.to(device)
133 return x, y
